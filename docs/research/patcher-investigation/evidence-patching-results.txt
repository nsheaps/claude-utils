=== KEY FINDING ===

The JS source code is EMBEDDED as plaintext strings within the Bun binary.
Binary string replacement WORKS if:
- Replacement is exactly the same byte length as original
- Padding uses valid JS (block comments /*...*/) or minimal whitespace
- Changes are conservative (small, targeted function body changes)
- The function structure is preserved (don't replace entire functions with
  radically different structures)

The patcher scripts CAN be updated to work on the binary by:
1. Using Python/binary tools instead of text-based readFileSync
2. Finding target strings with byte offsets instead of regex
3. Replacing with same-length padded replacements
4. Testing that the binary still runs after patching
